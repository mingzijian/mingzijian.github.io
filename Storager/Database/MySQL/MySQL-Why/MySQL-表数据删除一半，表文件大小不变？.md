# 删除数据

## 业务上 MysQL 常说的两种删除

### 逻辑删除

#### SQL 表现形式

update table_a  status = -1 where id = ?

### 物理删除

#### SQL 表现形式

delete from table_a where id = ?

在表文件上的表现



## 磁盘上的文件删除

不讨论回收站

### 逻辑删除（可恢复）

将文件所占的物理空间标记为已删除（可被重新写入数据覆盖）

### 物理删除（不可恢复）

Boom 物理摧毁

小文件填充覆盖



## 关联

MySQL 的物理删除 **近似于** 磁盘文件的逻辑删除。

即：delete 操作并不会对表文件进行物理删除，

配置中心
-------------

### 配置中心进化

```
服务实例1  ━━▶ 服务配置1（公共配置 + 服务1业务配置）
服务实例2  ━━▶ 服务配置2（公共配置 + 服务2业务配置）
服务实例3  ━━▶ 服务配置3（公共配置 + 服务3业务配置）



服务实例1  ┓  　　　　　┏　公共配置（全局配置）
服务实例2  ┣▶ 配置中心  ┫　服务1业务配置
服务实例3  ┛  　　　　　┃　服务2业务配置
　　　　              　　　  ┗　服务3业务配置
```

### 设计思想
- 集中管理 (UI/API)
- 环境区分 (dev/test/pre/prod)
- 动态刷新 (push/pull 批量刷新 灰度刷新)
- 变更审计 (review/rollback)
- 版本管理 (version)
- 弱依赖 （客户端做好配置缓存策略 ，同时配置中心本身也应尽可能减少对其他组件的依赖）
- 高可用 （集群）
- 快速上手（概念简单/SDK支持/框架集成）
